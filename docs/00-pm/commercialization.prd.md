# Executive Summary

BriefingRoom은 한국 정부 51개 부처 보도자료를 매일 자동 크롤링하여 AI로 요약하고 웹/텔레그램/RSS로 배포하는 정책 정보 플랫폼이다. 현재 공익 프로젝트로 운영 중이나, **정책 모니터링에 대한 강력한 B2B 수요**가 존재한다.

**핵심 결론**: 1인 개발자가 현실적으로 수익화할 수 있는 최적 경로는 **"금융권 규제 모니터링 SaaS"를 비치헤드로, 단계적으로 B2B 정책 인텔리전스 플랫폼으로 확장**하는 것이다.

- **Year 1 목표**: 금융 규제 모니터링 월 구독 (월 29만원) x 20개사 = ARR 약 7천만원
- **Year 2 목표**: 전 업종 확장 + API 서비스 = ARR 약 3억원
- **Year 3 목표**: 엔터프라이즈 + 데이터 라이선싱 = ARR 약 8억원

### Loop 1: Content-Led Growth
```

      -> 제품 인지
        -> 무료 체험
          -> 유료 전환
            -> 사례 연구 콘텐츠 생성
              -> (반복)
```

### Loop 2: Data Flywheel
```
더 많은 부처/기관 크롤링
  -> 더 풍부한 데이터
    -> AI 분석 품질 향상

        -> 구독 유지 + 입소문
          -> 매출 증가
            -> 크롤러 추가 투자
              -> (반복)
```

### Loop 3: B2B Referral
```
금융사 A 도입
  -> 성공 사례 공유 (금융 커뮤니티)
    -> 동종 금융사 관심

        -> 금융사 B 도입
```

## 5.2 Pre-mortem (사전 부검)

**시나리오**: "12개월 후 BriefingRoom 상업화가 실패했다. 왜 실패했을까?"

### Top 3 실패 원인 및 대응

**1. "B2B 영업을 하지 못해서 고객을 확보하지 못했다"**
- 확률: 높음
- 근본 원인: 1인 개발자가 제품 개발과 영업을 동시에 하기 어려움
- 사전 대응:
  - PLG(Product-Led Growth) 전략 채택 -> 셀프서비스 온보딩
  - 링크드인 콘텐츠 마케팅으로 인바운드 리드 확보
  - 무료 텔레그램 채널 -> 유료 전환 퍼널 자동화
  - Month 6까지 MRR 100만원 미달 시 피벗 의사결정

**2. "AI 요약 정확도가 기대 이하라 전문가가 신뢰하지 않았다"**
- 확률: 중간
- 근본 원인: qwen3.5:35b 모델의 한국어 금융 도메인 전문성 한계
- 사전 대응:
  - 금융 도메인 특화 프롬프트 최적화 (Few-shot + CoT)
  - 원문 링크를 항상 제공하여 "보조 도구"로 포지셔닝 (대체가 아닌 보완)
  - 베타 기간 전문가 피드백으로 정확도 개선
  - 필요시 Claude/GPT-4 등 상용 모델로 전환 검토

**3. "무료 사용자가 유료로 전환하지 않았다"**
- 확률: 중간-높음
- 근본 원인: 무료 기능이 충분하거나, 유료 기능의 가치를 체감하지 못함
- 사전 대응:
  - 무료: 당일 데이터만 / 유료: 과거 데이터 + 검색 + 알림 (명확한 차별)
  - 14일 체험 종료 전 "체험 리포트" 자동 발송 (얼마나 활용했는지)
  - 무료 -> 유료 전환 A/B 테스트 지속 실행
  - Persona별 차등 가격 (개인 9,900원 vs B2B 29만원)
