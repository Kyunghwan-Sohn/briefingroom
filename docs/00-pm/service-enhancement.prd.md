# govbrief.kr 서비스 강화 PRD

> Product Requirements Document
> 작성일: 2026-03-29
> 버전: 1.0
> 범위: 수익화 제외, 순수 서비스 품질 및 사용자 경험 강화

---

## Executive Summary

govbrief.kr은 대한민국 51개 정부 부처 보도자료를 자동 수집하고 AI로 요약하는 정책 정보 플랫폼이다. 현재 일 ~100건의 보도자료를 3줄 요약 + 5개 키워드로 제공하고 있으나, 정보의 "나열" 수준에 머물러 있어 사용자가 매일 재방문할 동기가 부족하다.

본 PRD는 **정보 소비 경험의 질적 전환**을 목표로, (1) 맞춤화를 통한 개인 관련성 극대화, (2) 맥락과 연결을 통한 정보 깊이 강화, (3) 시각화와 트렌드를 통한 탐색 경험 혁신을 핵심 전략으로 제안한다.

핵심 기능으로는 **개인 맞춤 피드 (관심 부처/키워드 구독)**, **정책 타임라인 (이슈 흐름 시각화)**, **주간 AI 인사이트 리포트**, **알림 시스템 (키워드 매칭)**, **북마크 및 읽기 이력**을 제안하며, 이를 통해 "정책 정보의 기본 소비처"에서 "정책 전문가의 필수 도구"로 포지셔닝을 전환한다.

## Part 1: Discovery (기회 발견)

### 1.1 5-Step Discovery Chain

#### Step 1: Brainstorm (가능성 탐색)

현재 govbrief.kr 사용자들의 잠재 니즈를 네 가지 사용자 군별로 도출한다.

**정책 담당자 (공무원/연구원)**
- "내 담당 분야 관련 타 부처 발표를 놓치지 않고 싶다"
- "특정 이슈가 여러 부처에 걸쳐 어떻게 진행되는지 한눈에 보고 싶다"
- "보도자료 원문 + 첨부 PDF를 빠르게 열어보고 싶다"
- "주간 동향을 상사에게 보고할 때 바로 쓸 수 있는 포맷이면 좋겠다"

**기자/미디어 종사자**
- "오늘 나온 보도자료 중 뉴스 가치가 높은 것을 먼저 보고 싶다"
- "특정 키워드(예: 반도체, 부동산)가 포함된 발표를 즉시 알림 받고 싶다"
- "과거 유사 보도자료와 비교해서 변화점을 파악하고 싶다"
- "관련 뉴스가 이미 얼마나 보도되었는지 알고 싶다"

**투자자/금융업 종사자**
- "금융 관련 규제 변경을 실시간에 가깝게 파악하고 싶다"
- "특정 산업(예: 2차전지, AI)에 영향을 줄 수 있는 정책을 추적하고 싶다"
- "숫자/수치가 포함된 보도자료를 우선적으로 보고 싶다"
- "정부 정책 트렌드가 시장에 미칠 영향을 예측하고 싶다"

**일반 시민**
- "나에게 직접 영향을 주는 정책(세금, 복지, 교육)만 골라보고 싶다"
- "정책 용어가 어려운데 쉽게 풀어서 알려주면 좋겠다"
- "관심 있는 정책의 진행 상황을 추적하고 싶다"
- "친구에게 공유할 때 깔끔한 카드 형태로 보내고 싶다"

#### Step 2: Assumptions (핵심 가정 식별)

| # | 가정 | 유형 |
|---|------|------|
| A1 | 사용자는 전체 보도자료가 아닌 "자신에게 관련된" 보도자료만 원한다 | 가치 |
| A2 | 단건 정보보다 이슈의 시계열 흐름을 보면 더 유용하다고 느낄 것이다 | 가치 |
| A3 | 키워드 기반 알림을 설정하면 재방문율이 높아질 것이다 | 성장 |
| A4 | AI 요약만으로는 부족하고, 맥락(관련 과거 발표, 관련 뉴스)이 필요하다 | 가치 |
| A5 | 북마크/읽기 이력이 있으면 업무 도구로 사용할 것이다 | 전환 |
| A6 | 시각적 데이터(차트, 트렌드 그래프)가 텍스트보다 정보 파악에 효과적이다 | UX |
| A7 | 소셜 공유 기능이 있으면 유기적 성장이 가능하다 | 성장 |
| A8 | 모바일에서의 읽기 경험이 데스크톱만큼 중요하다 | 실행 |

          |  A3(알림)      A6(시각화)
          |
중간 영향  |  A5(북마크)    A7(공유)
          |
낮은 영향  |  A8(모바일)
          |
          +------------------------------------------
             낮은 리스크     중간 리스크     높은 리스크

**우선순위 Top 3:**
1. A1 - 개인 맞춤화 (높은 영향, 낮은 리스크: localStorage로 구현 가능)
2. A3 - 알림 시스템 (높은 영향, 낮은 리스크: 기존 텔레그램 인프라 활용)
3. A4 - 맥락 정보 (높은 영향, 중간 리스크: 관련 뉴스 매칭 이미 존재)

### 2.3 SWOT Analysis

| | 긍정적 | 부정적 |
|---|--------|--------|
| **내부** | **Strengths** | **Weaknesses** |
| | - 51개 부처 통합 크롤링 인프라 | - Vanilla JS 단일 HTML (기술 부채) |
| | - AI 요약 파이프라인 확립 | - 서버리스 정적 사이트 (동적 기능 제한) |
| | - 텔레그램 채널 기반 사용자층 | - 사용자 인증/계정 시스템 없음 |
| | - 주간 PDF 리포트 자동 생성 | - 분석 도구 (Google Analytics 등) 미적용 |
| | - 금융 6개 서브카테고리 세분화 | - 과거 데이터 검색/조회 제한 |
| | - GitHub Actions 기반 자동화 | - 1인 운영 (콘텐츠 큐레이션 한계) |
| **외부** | **Opportunities** | **Threats** |
| | - 정부 투명성 관심 증가 | - korea.kr 자체 개편 가능성 |
| | - AI/LLM 기술 발전 (더 나은 요약) | - 빅카인즈 등 기존 서비스 확장 |
| | - PWA + 오프라인 지원 성숙 | - 정부 웹사이트 구조 변경 (크롤링 중단 리스크) |
| | - Canva 활용한 시각 콘텐츠 강화 | - API 과금 증가 (LLM 비용) |
| | - 정책 정보 민주화 수요 증가 | - 유사 서비스 등장 |

**SO 전략 (강점+기회):**
- AI 요약 인프라를 활용하여 더 깊은 분석 (트렌드, 연관 분석) 기능 추가
- 텔레그램 커뮤니티를 활용한 키워드 알림 시스템으로 리텐션 강화
- Canva 연동으로 공유용 정책 카드 이미지 자동 생성

**WT 전략 (약점+위협):**
- 정적 사이트 한계를 localStorage + IndexedDB로 보완 (서버 없이 개인화)
- 크롤링 중단 리스크에 대비하여 다중 소스 (korea.kr + 개별 부처) 유지
- 오픈소스/공익 포지셔닝으로 유사 서비스와 차별화

### 2.4 Strategic Positioning

**From**: "정부 보도자료 수집 + AI 요약 사이트" (정보 제공자)
**To**: "나에게 맞는 정책 정보를 매일 큐레이션하는 정책 인텔리전스 서비스" (의사결정 지원 도구)

핵심 전환 축:
1. 나열 → 큐레이션 (개인 맞춤)
2. 단건 → 맥락 (타임라인, 연관 분석)
3. 수동 방문 → 능동 전달 (알림, 뉴스레터)
4. 텍스트 → 시각화 (트렌드 차트, 정책 지도)

### 3.3 Market Sizing (비교 분석용 - 무료 서비스이므로 이용자 규모 추정)

**SAM (Serviceable Available Market) - 디지털 정책 정보 소비자:**
- 실제로 온라인으로 정부 보도자료를 주기적으로 확인하는 사람
- korea.kr 월간 방문자 수 기준 ~200만명 추정
- SAM: ~200만명

**SOM (Serviceable Obtainable Market) - 3년 내 확보 가능:**
- AI 요약 + 맞춤형 서비스에 반응할 얼리어답터
- 1차 (정책 전문가): 공무원 3만 + 기자 5천 + 금융업 2만 + 연구원 1만 = ~6.5만
- 2차 (정책 관심 시민): ~15만
- SOM: ~21.5만명 (3년 내 MAU 목표)

## Part 4: Go-To-Market (서비스 확산 전략)

### 4.1 ICP (Ideal Customer Profile)

govbrief.kr의 이상적 사용자는 "**업무상 정부 정책 변화를 일간 단위로 파악해야 하는 화이트칼라 전문가**"이다.

**ICP 특성:**
- 직업: 공무원, 기자, 애널리스트, 로비스트, 연구원
- 행동: 매일 아침 뉴스/정책 체크 루틴 보유
- 도구: 텔레그램/슬랙 활용, 모바일 우선, RSS 경험 있음
- 니즈: 정보 필터링 + 맥락 파악 + 빠른 원문 접근
- 가치관: 정보의 정확성과 시의성을 최우선시

### 4.2 Beachhead Segment (초기 집중 세그먼트)

**선정: "금융/경제 분야 전문가 (공무원 + 애널리스트 + 기자)"**

|------|-----------|------|
| 접근 가능성 | 5 | 이미 금융 6개 서브카테고리 구축, 텔레그램 채널 존재 |

| 구전 효과 | 5 | 금융권 네트워크 밀도 높음, "이 서비스 유용하다" 퍼지기 쉬움 |

### 4.3 GTM (Go-To-Market) Strategy - 유기적 성장

**Phase 1: 기반 구축 (Month 1-2)**
- 관심 분야 설정 + 키워드 알림 MVP 출시
- 텔레그램 채널에서 기존 사용자 대상 베타 테스트

**Phase 2: 콘텐츠 차별화 (Month 3-4)**
- 정책 타임라인 기능 출시
- 주간 AI 인사이트 리포트 품질 강화
- 공유용 카드 이미지 자동 생성 → SNS 확산

**Phase 3: 네트워크 확장 (Month 5-6)**
- 검색 고도화 (과거 데이터 전수 검색)

- 정책 관련 블로거/유튜버 협업

### 4.4 Competitive Battlecards

**vs korea.kr:**
- "korea.kr은 원문 저장소, govbrief.kr은 AI 큐레이션 레이어"
- 우리 강점: AI 3줄 요약, 카테고리 자동 분류, 모바일 UX, 키워드 알림
- 그들 강점: 공식 출처, 전체 데이터, 검색 API
- 전략: 보완 관계 포지셔닝 ("korea.kr 위에 얹는 스마트 레이어")

**vs 빅카인즈:**
- "빅카인즈는 뉴스 분석, govbrief.kr은 정부 발표 원문 분석"
- 우리 강점: 보도자료 원문 직접 처리, 무료, 일간 자동 업데이트, 단순한 UX
- 그들 강점: 고급 분석 도구, 대규모 뉴스 DB, 학술 활용
- 전략: 진입장벽 낮은 일상 도구 vs 전문 분석 도구로 차별화

**vs 직접 방문 (기본 습관):**
- "20개 사이트 방문하는 30분 vs govbrief.kr 3분"
- 우리 강점: 원스톱, AI 요약, 알림
- 전환 장벽: 습관 변경 필요, 데이터 정확성 신뢰 구축
- 전략: 점진적 전환 - 먼저 보조 도구로 시작 → 주요 도구로 전환

## Part 6: Pre-mortem & Risk Analysis

### 6.1 Pre-mortem (사전 실패 분석)

> "6개월 후 이 프로젝트가 실패했다면, 그 이유는 무엇일까?"

**Risk 1: 기술 부채 폭발 (가능성: 높음 / 영향: 높음)**
- 시나리오: 1,092줄 단일 HTML에 기능을 계속 추가하다 유지보수 불가능 상태 도달
- 완화: Phase 1에서 컴포넌트 기반 분리 (기능별 JS 모듈화) 선행, index.html을 빌드 스크립트로 생성하는 구조로 전환 검토
- 트리거: HTML 파일이 2,000줄 초과 시 리팩토링 착수

**Risk 2: 데이터 소스 중단 (가능성: 중간 / 영향: 치명적)**
- 시나리오: korea.kr 구조 변경 또는 크롤링 차단으로 데이터 수집 불가
- 완화: 개별 부처 크롤러(이미 7개 존재) 확장, 크롤링 실패 즉시 알림 체계, 데이터 백업
- 트리거: 특정 부처 크롤링 3일 연속 실패 시 대안 크롤러 개발

**Risk 3: 사용자 무반응 (가능성: 중간 / 영향: 높음)**
- 시나리오: 새 기능 출시해도 사용자가 관심을 보이지 않음
- 완화: 텔레그램 채널에서 사전 피드백 수집, 최소 기능 단위로 출시 후 반응 측정, A/B 테스트
- 트리거: 새 기능 출시 2주 후 사용률 5% 미만이면 피봇

### 7.2 Test Scenarios

#### TS-1: My Feed 설정 및 필터링

```
Given: 사용자가 govbrief.kr에 처음 방문
When:  관심 카테고리로 "금융경제"와 "산업기술"을 선택
And:   관심 키워드로 "반도체"를 등록
Then:  My Feed 탭에서 해당 카테고리 보도자료만 표시됨
And:   "반도체" 키워드가 포함된 보도자료가 하이라이트됨
And:   브라우저 새로고침 후에도 설정이 유지됨
```

#### TS-2: 정책 타임라인 표시

```
Given: 사용자가 금융위원회의 "가상자산 규제" 관련 보도자료 상세 보기
When:  상세 보기 하단 "관련 보도자료" 섹션 확인
Then:  동일 키워드를 공유하는 과거 보도자료 3-5건이 날짜순으로 표시됨
And:   각 관련 보도자료에 부처명, 날짜, 제목이 표시됨
And:   클릭 시 해당 보도자료 상세 보기로 이동
```

#### TS-3: 텔레그램 키워드 알림

```
Given: 사용자가 텔레그램 봇에서 /alert 부동산 명령 입력
When:  "부동산" 키워드가 포함된 보도자료가 크롤링됨
Then:  해당 사용자에게 텔레그램 메시지 발송 (제목 + 요약 1줄 + 링크)
And:   /alerts 명령으로 등록된 키워드 목록 확인 가능
And:   /remove 부동산 명령으로 삭제 후 해당 알림 중단
```

#### TS-4: 북마크 기능

```
Given: 사용자가 보도자료 카드를 보고 있음
When:  북마크 아이콘을 탭
Then:  아이콘이 채워진 상태로 변경 (시각적 피드백)
And:   "저장" 탭에서 북마크된 보도자료 목록 확인 가능
And:   다시 탭하면 북마크 해제
And:   다른 날짜로 이동 후 돌아와도 북마크 상태 유지
```

#### TS-5: 검색 고도화

```
Given: 사용자가 검색창에 "반도"를 입력
When:  자동완성이 활성화됨
Then:  "반도체", "반도체 수출", "한반도" 등 관련 키워드 제안
And:   검색 실행 시 제목 + 요약 + 키워드에서 매칭
And:   검색 결과에서 매칭 부분이 하이라이트됨
And:   날짜 필터를 추가하면 해당 기간 내 결과만 표시
```

#### TS-6: 다크 모드

```
Given: 사용자 기기의 시스템 설정이 다크 모드
When:  govbrief.kr을 방문
Then:  자동으로 다크 모드 UI 표시
And:   토글 버튼으로 라이트 모드 전환 가능
And:   선택이 localStorage에 저장되어 재방문 시 유지
```
