#### 아이디어 #1: FinLaw API -- 금융법령 검색 API

**한 줄 설명**: 핀테크/인슈어테크/웰스테크 스타트업이 자사 앱에 금융법령 검색을 임베드할 수 있는 REST API

**킬러 포인트**: "금융앱에 '근거 법령 보기' 버튼 하나 넣으려면 법제처 API 직접 연동 + 금융법 필터링 + 캐시 + 에러 핸들링을 다 해야 한다. 이 API는 `GET /v1/search?q=신용정보법+제32조` 한 줄이면 끝난다. 법제처 API의 CDN이다."

**수익 모델**:
- Free: 100 calls/day, 기본 메타데이터만
- Starter: 월 49,000원 -- 5,000 calls/day, 조문 본문 + 판례 연결
- Pro: 월 190,000원 -- 50,000 calls/day, AI 해설 + 해석례 + 헌재결정
- Enterprise: 협의 -- 전용 인스턴스, SLA 보장

**구현 난이도**: 하 | **예상 기간**: 2주
- finance_law.db를 읽기 전용 API로 래핑 (FastAPI/Flask)
- 법제처 API를 프록시하되 금융 도메인 필터 + 캐시 강화
- Cloudflare Workers 또는 Railway에 배포

**API 호출 예산**: 일 100~500회 (대부분 SQLite 로컬 히트, API는 캐시 미스 시에만)

#### 아이디어 #2: FinReg Monitor -- 금융 규제 변경 구독 SaaS

**한 줄 설명**: 금융사 컴플라이언스 담당자가 관심 법령/고시를 등록하면, 개정 즉시 diff + AI 영향 분석을 텔레그램/이메일/Slack으로 받는 구독 서비스

**킬러 포인트**: "금감원 제재를 받으면 최소 수억 과태료. 근데 금융위 고시가 바뀐 걸 3주 뒤에 알았다? 이 서비스는 변경 당일에 'OO고시 제X조 신설 -- 당신 회사에 이런 영향' 알림을 보낸다. 보험이다."

**수익 모델**:
- Basic: 월 99,000원 -- 법령 10개 워치, 일일 체크, 텔레그램 알림
- Standard: 월 290,000원 -- 법령 50개 + 고시/훈령 포함, AI 영향 분석, Slack 연동
- Premium: 월 590,000원 -- 무제한 + 주간 컴플라이언스 리포트 + API

**구현 난이도**: 중 | **예상 기간**: 4주
- 일 1회 cron: finance_law.db 조문 vs 법제처 API 최신 조문 비교
- 변경 감지 시 difflib으로 신구 대조 + LLM 영향 분석
- 텔레그램 기존 인프라 활용 + Slack webhook 추가

**API 호출 예산**: 일 200~500회 (워치리스트 법령 수 x 조문 수, 캐시로 절감)

#### 아이디어 #3: PolicyLaw Bridge -- 보도자료 x 법령 자동 연결 위젯

**한 줄 설명**: govbrief.kr의 금융 보도자료에 "이 정책의 법적 근거" 섹션을 자동 삽입하여, 보도자료 -> 조문 원문 -> 판례까지 원클릭 탐색

**킬러 포인트**: "금융위가 '가상자산 이용자 보호법 시행령 개정'을 발표했는데, 실제 조문이 뭐가 바뀌었는지는 보도자료에 안 나온다. 이 위젯이 조문 원문 + 신구 대조 + 관련 판례를 같이 보여준다. 정책 브리핑이 법률 분석으로 업그레이드된다."

**수익 모델**:
- govbrief.kr 무료 사용자: 보도자료 + 법령 링크 (광고 모델)
- 프리미엄 구독: 월 9,900원 -- AI 법령 해설 + 판례 연결 + PDF 다운로드
- 기관 구독: 월 99,000원 -- 팀 공유 + 커스텀 알림 + API

**구현 난이도**: 하 | **예상 기간**: 1주
- 기존 law.py의 process_law_for_item() 이미 보도자료-법령 연결 구현 완료
- 정적 사이트 템플릿에 법령 카드 UI 추가
- finance_law.db에서 조문 본문 직접 조회 (API 불필요)

**API 호출 예산**: 일 0~50회 (대부분 finance_law.db 로컬 히트)

**구현 난이도**: 중 | **예상 기간**: 3주

**API 호출 예산**: 일 50~100회 (법령 연혁 조회 시)

- Standard: 월 190,000원 -- 체크리스트 생성 + 변경 알림 + 제재 사례 연결
- Enterprise: 월 490,000원 -- 팀 관리 + 감사 이력 + 커스텀 카테고리 + API

**구현 난이도**: 중 | **예상 기간**: 5주

**수익 모델**:
- 무료: 월 1건 기본 스캔
- Pro: 월 290,000원 -- 무제한 스캔 + AI 리스크 분석 + 제재 사례 연결
- Enterprise: 건당 50만원 -- 심층 분석 리포트 (LLM + 수동 검토 병행)

> 금융사 컴플라이언스 담당자가 워치리스트에 등록한 법령/고시/훈령이 개정되면, 변경 당일 신구 대조 + AI 영향 분석을 텔레그램/Slack/이메일로 발송하는 B2B 구독 서비스

### 사용 시나리오

```
[Day 1] 컴플라이언스 담당자 김과장이 가입
  -> 업무 영역 선택: "여신"
  -> 시스템이 자동 추천: 여신전문금융업법, 대부업법, 이자제한법 등 12개 법령
  -> 추가 등록: 금감원 여신심사 모범규준, 금융위 고시 3건
  -> 알림 채널 설정: Slack #compliance 채널 + 이메일

[Day 15] 금융위가 "여신전문금융업법 시행령 일부 개정령" 공포
  -> 시스템이 일일 체크에서 변경 감지
  -> 자동 처리:
     1. 개정 전 조문 (finance_law.db) vs 개정 후 조문 (API 최신) diff 생성
     2. LLM이 변경 사항 요약: "카드론 금리 산정 기준 변경 (제X조)"
     3. LLM이 영향 분석: "카드사 여신 시스템 금리 산정 로직 수정 필요. 시행일: 6개월 후"
     4. Slack으로 알림 발송 + 웹 대시보드 업데이트

[Day 16] 김과장이 대시보드에서 확인
  -> 신구 대조표 (빨간색=삭제, 초록색=추가, 노란색=변경)
  -> AI 영향 분석 리포트
  -> "보고서 생성" 클릭 -> 내부 보고용 PDF 자동 생성
  -> 체크리스트에 "시스템 수정 필요" 액션 아이템 추가
```

**핵심 컴포넌트**:

1. **변경 감지 엔진** (`finreg_monitor.py`)
   - finance_law.db의 조문 해시값 vs 법제처 API 최신 조문 해시값 비교
   - 변경 시 difflib.unified_diff()로 신구 대조 생성
   - 변경 내역을 monitor.db에 기록

2. **LLM 영향 분석** (`finreg_analysis.py`)
   - 변경된 조문 + 관련 판례/해석례를 컨텍스트로 제공
   - 프롬프트: "이 법령 개정이 금융회사 실무에 미치는 영향을 분석하세요"
   - 영향도 스코어 (상/중/하) + 시행일까지 남은 일수 자동 계산

3. **알림 발송** (기존 telegram.py 확장)
   - Slack Incoming Webhook 추가
   - 이메일: SendGrid 무료 티어 (일 100건)

4. **웹 대시보드** (GitHub Pages)
   - 정적 HTML + JavaScript로 대시보드 렌더링
   - 로그인: GitHub OAuth 또는 단순 토큰 인증
   - PDF 생성: 서버사이드 (pdfkit) 또는 클라이언트 (jsPDF)

**데이터 모델** (monitor.db 추가):

```sql
-- 사용자 워치리스트

    id INTEGER PRIMARY KEY AUTOINCREMENT,

    id INTEGER PRIMARY KEY AUTOINCREMENT,

    sent_at TEXT,
    read_at TEXT
);
```

## TOP 2: FinLaw GPT -- 금융법 특화 AI 어시스턴트

### 서비스 정의

> finance_law.db 5,613건 전체를 RAG 컨텍스트로 장착하여, 금융법에 대한 모든 질문에 조문 원문 + 판례 + 해석례 근거를 포함한 정확한 답변을 제공하는 AI 어시스턴트. 텔레그램 봇 + 웹 + ChatGPT 커스텀 GPT 3채널 동시 운영.

```python
def render_law_widget(laws: list[dict]) -> str:
    """보도자료 페이지에 삽입할 법령 카드 HTML 생성"""
    if not laws:
        return ""

    html = '<div class="law-widget"><h3>관련 법령</h3>'
    for law in laws:
        # finance_law.db에서 직접 조회 (API 불필요)
        articles = get_articles_from_db(law["law_name"])
        cases = get_cases_from_db(law["law_name"])

        html += f'<div class="law-card">'
        html += f'<h4>{law["law_name"]}</h4>'
        for art in articles[:3]:
            html += f'<p class="article">{art["article_no"]} {art["title"]}</p>'
            html += f'<p class="content">{art["content"][:200]}...</p>'
        if cases:
            html += f'<p class="cases">관련 판례 {len(cases)}건</p>'
        html += '</div>'

    html += '</div>'
    return html
```
