## 5. 기술 분석 요약

### 렌더링 방식별 분류

| 방식 | 사이트 | 크롤링 방법 |
|------|--------|-------------|
| **SSR (서버사이드)** | MSS, SMES, NIPA, KOCCA, KEITI, MOHW, MOEL, KISED, MOE, MOIS | requests + BeautifulSoup |
| **SSR + AJAX 하이브리드** | NTIS, K-Startup, MOTIE | requests + API 엔드포인트 분석 |
| **CSR (클라이언트)** | IITP (Vue.js), NRF (AJAX) | Playwright/Selenium 또는 API 직접 호출 |
| **접속 불안정/인증** | G2B, KETEP, SEMAS | 별도 전략 수립 필요 |

### 공통 HTML 패턴

대부분의 정부 사이트가 다음 패턴을 따름:
- **게시판 프레임워크**: 전자정부 표준프레임워크(eGovFrame) 기반
- **테이블 구조**: `<table>` 태그 내 `<thead>` + `<tbody>` 표준 구조
- **페이지네이션**: 숫자 링크, `pageIndex` 또는 `curPage` 파라미터
- **상세 URL**: `?seq=` 또는 `?bcIdx=` 등 시퀀스 번호 기반

### 주의사항

1. **User-Agent 필수**: 일부 사이트(MSS, MOTIE)는 봇 차단 가능
2. **요청 간격**: 초당 1회 이하 권장 (robots.txt 확인)
3. **도메인 변경**: MOTIE(motie->motir), ME(me->mcee) 등 부처 개편에 따른 URL 변경 주시
4. **중복 제거**: MSS <-> SMES, K-Startup <-> KISED 간 동일 공고 중복 게시 빈번
5. **IRIS 연계**: R&D 과제는 부처 홈페이지에 공고 후 IRIS에서 접수 -> IRIS도 크롤링 대상 고려
