## 1. 현황 진단 (As-Is)

### 기술 스택 확인 결과

- `index.html` 단일 파일 정적 사이트 (GitHub Pages, 서버 비용 0원)
- `data/*.json` 정적 파일 fetch → 클라이언트 사이드 렌더링
- 5개 분야 분류: 금융경제 / 사회복지 / 산업기술 / 외교안보 / 행정법제
- 금융기관 세부 분류 7개 서브카테고리 (FIN_SUB_MAP 구현됨)
- PWA manifest, RSS feed 구현됨
- 모바일 바텀시트 패턴, 반응형 완성
- Google AdSense `ads.txt` 설치됨 (수익 미검증)
- 텔레그램 채널 @govbriefkr 운영 중 (일 2회 브리핑)
- 주간 리포트 기획 완료 (docs/weekly-content-plan.md)

### 핵심 자산 재평가

| 자산 | 현재 활용도 | 잠재가치 |
|------|-------------|----------|
| 51개 부처 일일 150건+ 자동수집 | 운영 중 | 데이터 API 판매 가능 |
| AI 요약 + 키워드 추출 | 웹 표시만 | 뉴스레터/리포트 콘텐츠화 가능 |
| 5개 분야 + 7개 금융 서브카테고리 분류 | 필터 UI만 | 개인화 / 알림 트리거 가능 |
| 주간 집계 데이터 | 미활용 | 인텔리전스 리포트 판매 가능 |
| 텔레그램 채널 | 무료 브리핑 | 프리미엄 채널 분리 가능 |

### 문제 인식

1. **수익 구조 단일화 위험**: AdSense만으로는 고품질 타겟 트래픽 가치를 회수 불가
2. **무료 혜택의 역설**: 공무원/기자/금융인이 쓰는 B2B급 서비스를 무료 제공 중
3. **이탈 방지 장치 없음**: 읽음 처리, 북마크, 히스토리 없어 재방문 동기 약함
4. **리드 수집 채널 없음**: 이메일 구독 CTA 없어 직접 마케팅 불가

## 3. 수익화 모델 설계

### 3-1. 수익원 포트폴리오 (MoSCoW)

#### Must — 즉시 실행 (0~4주)

**A. 이메일 뉴스레터 (무료 → 유료 전환 퍼널 입구)**

- 도구: Stibee 또는 Substack (국내 세금계산서 발행 고려 시 Stibee 권장)
- 무료 티어: 매일 오전 7시, 분야별 TOP 5 요약 (AI 생성)
- 구독 CTA: 헤더 우측 `header-sub-area` 영역을 텔레그램 → 이메일 구독으로 교체하거나 병렬 배치
- 목표: 첫 3개월 1,000명, 6개월 3,000명

**구현 방법:**

```
기존 헤더:
  [텔레그램 채널] 버튼 단독

변경 후:
  [이메일 구독] 버튼 (primary)  [텔레그램] 버튼 (secondary)
```

현재 `header-sub-btn` 클래스는 이미 스타일 완성됨. href만 Stibee 구독 폼 URL로 교체하면 즉시 가동.

**B. AdSense 최적화**

현재 ads.txt는 있으나 실제 광고 삽입 코드가 index.html에 보이지 않음. 즉시 삽입 위치:
- 카드 그리드 10번째 아이템 위치에 네이티브 광고 블록 삽입
- 상세 모달 하단 (d-actions 아래)에 인-아티클 광고
- 타겟 유저(공무원/기자/금융인)는 일반 CPM보다 2~3배 단가 높음

#### Should — 1~3개월

**C. 프리미엄 구독 (월 3만원 / 연 25만원)**

유료화할 기능의 핵심 원칙: **무료 사용자가 매일 쓰는 기능은 그대로 무료, 업무 효율을 올리는 기능은 유료.**

| 기능 | 무료 | 유료 |
|------|------|------|
| 당일 보도자료 조회 | O | O |
| AI 요약 | O | O |
| 검색 | O | O |
| 30일 이상 아카이브 조회 | X | O |
| 관심 부처/분야 설정 → 맞춤 피드 | X | O |
| 푸시 알림 (특정 부처/키워드) | X | O |
| 북마크 + 폴더 | X | O |
| 월간 인텔리전스 리포트 PDF | X | O |
| CSV 다운로드 (기간별 보도자료 목록) | X | O |

**구현 방식 — 서버리스로 최소화:**
- 인증: Clerk.dev 무료 티어 (소셜 로그인, 월 1만 MAU까지 무료)
- 결제: 토스페이먼츠 또는 PortOne (구 아임포트) + 정기결제
- 데이터 저장(북마크/설정): Supabase 무료 티어 (500MB, 충분)
- GitHub Pages 유지 + API는 Supabase Edge Functions 또는 Cloudflare Workers

**D. 프리미엄 텔레그램 채널 분리**

- 무료 채널 @govbriefkr: 현행 유지 (일 2회, 분야별 Top)
- 유료 채널 @govbriefkr_pro: 월 1만 5천원
  - 실시간 알림 (보도자료 등록 30분 내)
  - 특정 부처 단독 채널 (예: 금융위 전용 브리핑)
  - 주간 심층 분석 메시지 (현재 weekly-content-plan.md 기획 그대로)
- 운영 방식: Telegram 유료 채널 기능 또는 Combot/Telestats + 인바이트 링크 유료화

#### Could — 3~6개월

**E. B2B 데이터 API**

- 대상: 법무법인, 로비스트, 컨설팅 펌, 증권사 리서치팀, 싱크탱크
- 제공: REST API — `/api/v1/press-releases?date=&ministry=&category=&limit=`
- 요금: 월 30만원 (1,000 req/day) / 월 100만원 (무제한)
- 기술: Cloudflare Workers + R2로 현재 data/*.json 서빙 구조 그대로 API화
- 계약: NDA + 데이터 이용 약관 필수

**F. 부처별 정기 인텔리전스 리포트**

- 형태: 월 1회, 부처별 또는 분야별 PDF 리포트 (8~12페이지)
- 내용: 이번 달 핵심 이슈, 키워드 트렌드, 전월 대비 변화, 주목 정책 3선
- 요금: 건당 10만원 / 구독 월 30만원 (특정 부처 2개 선택)
- 타겟: 해당 분야 로비스트, 컨설팅사, 학술 연구자
- 생성: 기존 AI 요약 파이프라인 확장 (월말 집계 → GPT 리포트 생성 → PDF)

#### Won't — 현재 범위 외

- 자체 뉴스 생산 (기자 고용): 콘텐츠 책임 리스크 + 비용 과다
- 광고 네트워크 직판 세일즈: 운영 부담 대비 수익 낮음 (트래픽이 100만/월 이상 되기 전)
- 해외 서비스 확장: 수요 검증 전 자원 낭비

## 4. 모바일 정보 제공 최적화

### 4-1. 출퇴근 3분 시나리오

타겟 유저의 핵심 컨텍스트: 지하철/버스, 서서 한 손으로 스마트폰, 3분 이내.

**현재 모바일 UX 분석 (index.html 기준):**
- `mob-filter` 가로 스크롤 칩 구현됨 — 분야 선택 가능
- `mobile-date-nav` 날짜 이동 구현됨
- `detail` 모달이 바텀시트로 올라오는 패턴 구현됨
- `mobile-cta` 고정 하단 바 구현됨 (현재 텔레그램 CTA)

**필요한 추가 개선 (우선순위 순):**

1. **"오늘의 핵심" 3줄 요약 카드** (최상단 고정)
   - 현재 `top-picks` div가 존재하나 `display:none` 상태
   - 매일 오전 AI가 전체 보도자료 중 중요도 상위 3건 선정 → 별도 JSON 필드로 관리
   - 구현: `data/top-picks.json` 생성 → `loadData()` 시 병렬 fetch

2. **읽음 처리** (localStorage 기반, 서버 불필요)
   - 카드 클릭 시 `localStorage.setItem('read_' + id, true)`
   - 읽은 카드에 시각적 dim 처리 (opacity 0.5 또는 체크 아이콘)
   - 구현 난이도: 낮음 (JS 10줄 + CSS 1줄)

3. **북마크** (로그인 없는 무료 버전은 localStorage, 유료는 Supabase)
   - 카드 하단 북마크 버튼 추가
   - 사이드바/모바일 필터에 "북마크" 탭 추가

4. **"어제 놓친 것" 배지**
   - 전날 미읽음 건수를 앱 아이콘 배지 + 상단에 표시
   - PWA badge API 활용 (`navigator.setAppBadge(count)`)

### 4-2. 푸시 알림 전략

**채널별 특성과 권장 전략:**

| 채널 | 도달률 | 구현 비용 | 권장 용도 |
|------|--------|-----------|-----------|
| 텔레그램 | 높음 (앱 알림) | 완성됨 | 일반 정기 브리핑 |
| PWA Push | 중간 (iOS 16.4+ 필요) | 중간 | 긴급/속보 알림 |
| 이메일 | 낮음 (오픈율 25~35%) | 낮음 | 일일 다이제스트 |

**권장 구현 순서:**

1. 이메일 구독 즉시 가동 (2주 내) — 리드 수집이 최우선
2. 텔레그램 프리미엄 채널 (4주 내) — 기존 인프라 활용
3. PWA Push (3개월 후) — 유료 구독자 전용 기능으로 차별화

**PWA Push 구현 방법:**
- Cloudflare Workers + Web Push Protocol (서버 필요)
- 또는 OneSignal 무료 티어 (월 1만 구독자까지 무료)
- 트리거: 금융위/기재부 등 특정 부처 보도자료 등록 시 → 해당 부처 알림 설정 유저에게 발송

### 4-3. 개인화 피드

**설계 원칙:** 서버 없이 시작, 유료 전환 시 서버 동기화.

**무료 개인화 (localStorage):**
```javascript
// 관심 부처 저장
localStorage.setItem('watchlist_ministries', JSON.stringify(['금융위원회','기획재정부']))
// 관심 분야 저장
localStorage.setItem('watchlist_categories', JSON.stringify(['금융경제']))
// 로드 시 watchlist 기반으로 기본 필터 적용
```

**유료 개인화 (Supabase):**
- 로그인 시 설정 클라우드 동기화
- 여러 기기에서 동일 설정 유지
- 관심 키워드 추가 (예: "AI", "탄소중립") → 매칭 보도자료 하이라이트

## 5. 핵심 사용자 여정 (수익화 연결)

### 5-1. 전환 퍼널

```
[검색/SNS 유입] → [무료 웹 사용] → [이메일 구독] → [프리미엄 전환]
      |                  |                |                   |
   SEO/텔레그램        3일 습관화        뉴스레터 가치       업무 효율 체감
   공유 트래픽         경험 제공         검증 단계           전환 결정
```

**1단계: 유입 (현재 상태 유지 + 개선)**
- SEO: 각 날짜별 URL (`/?date=2026-03-28`) + 부처별 페이지 → 구글 인덱싱
- 텔레그램 → 웹사이트 트래픽 전환 (기존 메시지에 govbrief.kr 링크 포함)

**2단계: 무료 웹 사용 → 이메일 구독 전환**

전환 트리거 3가지:

| 트리거 | 시점 | 구현 |
|--------|------|------|
| 3건 이상 상세 열람 시 | 3번째 모달 열릴 때 | 모달 하단에 "이 보도자료, 내일도 받아보세요" CTA |
| 첫 방문 30초 후 | 타이머 | 현재 `hero-banner` 영역 활용 (dismiss 가능) |
| 모바일 CTA | 스크롤 하단 | 현재 `mobile-cta` div 활용 (텔레그램 → 이메일로 교체 또는 병렬) |

**3단계: 이메일 구독 → 프리미엄 전환**

- 무료 구독자에게 매주 1회 프리미엄 기능 소개 (북마크한 자료 CSV 다운로드, 30일 아카이브)
- "이번 달 금융위 동향 PDF 미리보기" → 전문 보려면 프리미엄 업그레이드
- 전환 목표: 이메일 구독자의 3~5% → 월 3,000명 구독자 기준 90~150명 유료화

**4단계: 개인 유료 → B2B 업셀**

- 유료 구독 중인 법인 도메인 이메일 사용자 식별
- 개인 구독료의 10배 수준 B2B 계약 제안 (팀 계정 + API 접근)

### 5-2. 전환 단계별 제공 가치

```
무료 웹          이메일 구독      프리미엄(3만원)   B2B API(30만원~)
─────────────────────────────────────────────────────────────────
오늘 보도자료   일일 TOP5 요약   무제한 아카이브   원시 데이터 API
AI 요약 열람    오전 7시 발송    관심 부처 알림     부처별 통계 API
분야 필터       주간 리포트      북마크+폴더        월간 인텔리전스
텔레그램 링크   무료 기능 유지   CSV 다운로드       SLA 보장
```

## 6. 경쟁 서비스 대비 차별화

### 6-1. 경쟁 현황 분석

| 서비스 | 강점 | 약점 |
|--------|------|------|
| 정책브리핑(korea.kr) | 공식 출처 | UI 복잡, 검색 느림, 요약 없음 |
| 각 부처 직접 | 1차 출처 | 51개 사이트 개별 접속 필요 |
| 뉴스통신사(연합뉴스 등) | 해석/분석 포함 | 유료 구독, 정치적 편향 가능 |
| 빅카인즈 | 아카이브 강함 | 언론사 기사 중심, 보도자료 아님 |

### 6-2. govbrief.kr 고유 가치 제안

**핵심 차별점 3가지:**

1. **"원문 그대로" 신뢰성 + AI 요약의 속도**
   - 뉴스통신사: 기자의 해석 개입 → 편향 가능
   - govbrief.kr: 정부가 발표한 원문 그대로 + AI 중립 요약
   - 메시지: "기자의 필터 없이, 정부 발표를 직접 3줄로"

2. **금융기관 특화 분류 (경쟁 서비스 없음)**
   - 한국은행/금융위/금감원/KDB 등 7개 서브카테고리 분류
   - 금융인이 korea.kr에서 이걸 찾으려면 기관별 사이트 7개를 돌아야 함
   - govbrief.kr은 "금융 → 감독·규제" 클릭 한 번

   - "이번 주 AI 관련 보도자료 150% 급증" 같은 인사이트
   - 어느 경쟁 서비스도 제공하지 않음
